# Upload Workflow Guide

Use this workflow for the remaining uploads. The goal is to keep the upload process simple now and organize everything later after all content is in the system.

## 1. What to do first

- Do not re-upload the remaining five Old Testament books.
- We will migrate those books using the same migration process used for the previous 34 books.
- After migration, the entries will appear in the admin panel.
- Titles can be corrected later in the admin panel if needed.

## 2. What to upload now

Continue uploading the remaining content one by one:

- New Testament videos
- 53 Bedtime Stories

Use the same upload flow you have already been following.

## 3. Upload screen fields

Open the admin panel and go to the video upload section.

### Fields to use

```text
Video Info
------------------------------------------------
Title            -> enter the video title
Category         -> Kids Videos
Video File       -> select the video file
Vertical Banner  -> upload the cover/banner image
Subtitle File    -> optional
Order Index      -> leave empty
Published        -> keep as needed
Locked           -> keep as needed
------------------------------------------------
```

### Important field rules

- Category must be `Kids Videos`
- Leave `Order Index` empty
- Upload videos one by one
- Do not create separate categories such as:
  - Kids Old Testament
  - Kids New Testament
  - Kids Bedtime Stories

## 4. Recommended workflow

1. Migrate the remaining five Old Testament books.
2. Continue uploading the New Testament videos.
3. Upload the 53 Bedtime Stories.
4. Keep using `Kids Videos` as the category.
5. Leave `Order Index` empty while uploading.
6. After all uploads are complete, we will organize the content, assign categories where needed, and finalize display order.

## 5. Admin panel follow-up

Once all uploads are finished:

- Update titles for the migrated Old Testament books if necessary.
- Organize content into the correct categories.
- Configure the final display order.
- Perform a final verification in both the admin panel and the mobile application.
